# AI Native 多智能体协作平台演示脚本

目标时长：8-10 分钟。

## 1. 启动与就绪检查

```bash
docker compose up -d --build api api-daprd worker worker-daprd mcp-server mcp-server-daprd web redis postgres placement scheduler jaeger prometheus
curl http://localhost:8000/health
curl http://localhost:8000/ready
curl http://localhost:8000/api/tools
```

讲解点：

- Compose 同时启动 API、Worker、MCP Server、Web、Dapr sidecar、Redis、PostgreSQL、Jaeger 和 Prometheus。
- `/ready` 检查 Redis、PostgreSQL、Dapr sidecar 和 MCP 工具发现。
- `/api/tools` 动态发现 4 个 MCP 工具：calculator、web_search、code_runner、readonly_sql。

## 2. Web 控制台演示

浏览器打开 `http://localhost:5173`。

1. 在“新建任务”输入研究主题。
2. 分别选择 `LangGraph`、`CrewAI` 或 `自动`。
3. 提交后进入任务详情页。
4. 展示状态、实际引擎、选择原因、Agent 时间线、工具调用、Token 指标和最终报告。
5. 刷新详情页，说明任务状态来自 PostgreSQL 与 Dapr State，不依赖前端内存。

## 3. LangGraph 路径

命令：

```bash
curl -X POST http://localhost:8000/api/tasks \
  -H "Content-Type: application/json" \
  -d '{"user_query":"用三点概括 Dapr Workflow 的作用","engine":"langgraph","delay_seconds":0}'
```

讲解点：

- Workflow 调度后由 Worker 执行。
- LangGraph 使用 `StateGraph`，节点为 `plan -> select_roles -> researcher -> analyst -> writer -> persist_result`。
- `thread_id=task_id`，checkpoint 由 Dapr State adapter 保存。

## 4. CrewAI 路径

命令：

```bash
curl -X POST http://localhost:8000/api/tasks \
  -H "Content-Type: application/json" \
  -d '{"user_query":"整理 LangGraph 与 CrewAI 的适用场景","engine":"crewai","delay_seconds":0}'
```

讲解点：

- CrewAI 使用共享角色配置构建 Agent/Task/Crew。
- Workflow 将 researcher、analyst、writer 拆为独立 Activity，失败时只重试对应角色步骤。
- 输出 schema 与 LangGraph 对齐，前端和数据库不需要区分内部框架。

## 5. 自动路由

命令：

```bash
curl -X POST http://localhost:8000/api/tasks \
  -H "Content-Type: application/json" \
  -d '{"user_query":"请多角色协作完成一份 Dapr Agents 调研报告","engine":"auto","delay_seconds":0}'
```

讲解点：

- `auto` 调用 LLM 生成结构化 `{engine, reason, subtasks}`。
- 结构化解析失败时 fallback 到 LangGraph。
- 手动选择的 `langgraph` 或 `crewai` 不会被自动路由覆盖。

## 6. 暂停、恢复与故障恢复

命令：

```bash
TASK_ID=$(curl -sS -X POST http://localhost:8000/api/tasks \
  -H "Content-Type: application/json" \
  -d '{"user_query":"recovery demo","engine":"langgraph","delay_seconds":30}' | jq -r .task_id)

curl -X POST http://localhost:8000/api/tasks/$TASK_ID/pause
curl -X POST http://localhost:8000/api/tasks/$TASK_ID/resume
docker compose stop worker
docker compose start worker
curl http://localhost:8000/api/tasks/$TASK_ID
```

讲解点：

- Workflow instance ID 与 task ID 可关联。
- Activity 使用幂等键避免重复业务记录。
- Worker 重启后任务继续或安全重试。

## 7. 可观测性

打开：

- Jaeger: `http://localhost:16686`
- Prometheus: `http://localhost:9090`

讲解点：

- Jaeger 通过 `trace_id` 关联 API、Workflow Activity、Agent 节点和 MCP 工具调用。
- Prometheus 查询示例：

```promql
task_completions_total
task_duration_seconds_bucket
llm_tokens_total
tool_calls_total
```

## 8. 一键验收脚本

快速检查：

```bash
uv run python scripts/day10_acceptance.py --quick
```

完整检查：

```bash
uv run python scripts/day10_acceptance.py --engines langgraph,crewai,auto --concurrency 10 --output docs/day10-acceptance-results.json
```

脚本输出可作为最终报告的测试证据。
