# explanation

请基于当前 Git 未提交的改动（`git status` + `git diff`），逐文件介绍本次变更做了什么、改变了什么，帮助我理解和 review 代码。

要求：
- 先给 2–4 句整体摘要：改动目的、涉及模块、主要数据流或行为变化。
- 再按文件逐一说明（仅覆盖新增/修改的文件；删除或重命名单独标注）：
  - 该文件在系统中的角色
  - 具体改了什么（新增函数/类、逻辑调整、接口或契约变化等）
  - 与其他文件的关联（谁调用、谁被影响）
  - 若有非显而易见的设计取舍，简要说明原因
- 用中文，偏说明而非挑错；不要重复贴大段 diff，引用关键代码时用简短片段即可。
- 若改动彼此独立，按模块分组；若是一条链路，按执行顺序说明。
- 最后列出 3–5 条 review 时最该关注的点（行为变化、边界、测试缺口），不替代正式 code review。
